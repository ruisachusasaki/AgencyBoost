CONTEXT:
This is AgencyBoost, a CRM for marketing agencies. We are building a 
NEW HIRE WORKFLOW feature. The following has already been completed:
  - DB tables: ic_agreement_templates, job_offers, offer_signatures, 
    offer_status_log
  - New status values on applicants table: 'send_offer', 
    'offer_accepted', 'offer_declined', 'hired'
  - IC Agreement template builder under Settings > HR Settings > 
    IC Agreement

This prompt builds the SEND OFFER FLOW — when a manager sets an 
applicant's status to "Send Offer", a modal opens to collect offer 
details, AgencyBoost populates the IC Agreement template, generates 
a unique signing link, auto-emails it to the applicant, and shows 
the manager a copyable link.

ALWAYS use the primary teal theme color (hsl(179, 100%, 39%) / #00C9C6) 
for buttons, accents, and interactive elements. Never use blue.

PART A — BACKEND API ROUTES
═══════════════════════════════════════

Add the following routes to server/routes.ts. All require 
requireAuth(). Admin and Manager roles only unless noted.

─────────────────────────────────────
POST /api/applications/:applicationId/send-offer
─────────────────────────────────────
Purpose: Creates a job_offer record, populates the IC Agreement 
template with real values, generates a signing token, sends the 
signing link email to the applicant.

Body:
  {
    compensation: string,
    compensationType: 'per_hour' | 'per_month' | 'flat_rate',
    startDate: string (ISO date),
    customTerms: string (optional, nullable)
  }

Logic:

STEP 1 — Validate application
  - Fetch the application by applicationId
  - If not found: return 404
  - If application already has a job_offer record (check 
    job_offers where applicationId = this id): return 400 
    with message "An offer has already been sent for this 
    application."
  - If application status is already 'offer_accepted', 
    'offer_declined', or 'hired': return 400 with message 
    "This application is no longer in an offerable state."

STEP 2 — Fetch active IC Agreement template
  - Query ic_agreement_templates where isActive = true
  - If none found: return 400 with message "No active IC 
    Agreement template found. Please configure one under 
    Settings > HR Settings > IC Agreement before sending 
    offers."

STEP 3 — Populate template content
  - Take template.content (HTML string with {{placeholders}})
  - Replace ALL placeholders with real values:
      {{candidate_name}}    → application's full name 
                              (check actual column name)
      {{position}}          → application's position
      {{start_date}}        → startDate from request body, 
                              formatted as "March 13, 2026"
      {{compensation}}      → compensation from request body
      {{compensation_type}} → human-readable version:
                              'per_hour' → 'per hour'
                              'per_month' → 'per month'
                              'flat_rate' → 'flat rate'
      {{offer_date}}        → today's date formatted as 
                              "March 13, 2026"
      {{custom_terms}}      → customTerms from body, or 
                              empty string if null
      {{manager_name}}      → req.user's full name
      {{company_name}}      → "The Media Optimizers"
  - Store the fully populated content as populatedContent
  - This is the permanent legal record — immutable after creation

STEP 4 — Generate signing token
  - Generate a cryptographically secure random UUID using 
    Node.js crypto.randomUUID()
  - This becomes the signingToken
  - The public signing URL will be: 
    {APP_BASE_URL}/sign-offer/{signingToken}
  - APP_BASE_URL: read from environment variable APP_BASE_URL 
    or VITE_APP_URL — check which env var holds the base URL 
    in the existing codebase. Fall back to 
    'https://agencyboost.app' if not found.

STEP 5 — Create job_offer record (in a transaction):
  - Insert into job_offers:
      applicationId, createdBy = req.user.id, 
      templateId = template.id,
      populatedContent, compensation, compensationType,
      startDate, customTerms, signingToken,
      status = 'pending', sentAt = now()
  - Insert into offer_status_log:
      offerId = new offer id, status = 'pending',
      changedBy = req.user.id,
      note = 'Offer created and sent'

STEP 6 — Update application status
  - Update the application record: status = 'send_offer'
  - Use the actual column and table names from the existing 
    applicants table

STEP 7 — Send signing link email to applicant
  - Use the existing email sending pattern in the codebase 
    (check how other emails are sent — Nodemailer, SendGrid, 
    or similar — use the SAME pattern)
  - To: application's email address
  - Subject: "Your offer from The Media Optimizers — 
    Action Required"
  - Body (HTML email):
      Greeting: "Hi [candidate_name],"
      
      Paragraph: "We're excited to extend this offer to you 
      for the [position] role at The Media Optimizers. Please 
      review your Independent Contractor Agreement and sign 
      it at the link below."
      
      CTA Button (teal): "Review & Sign Your Agreement"
      href: the full signing URL

      Paragraph: "This link is unique to you. Please do not 
      share it. If you have any questions, reply to this email 
      or contact your hiring manager directly."
      
      Signature: "The Media Optimizers Team"
  - If email sending fails: do NOT roll back the offer 
    creation — the offer is still valid. Log the error and 
    return a success response with a warning flag:
      { success: true, signingUrl: '...', emailSent: false, 
        warning: 'Offer created but email failed to send. 
        Please share the link manually.' }
  - If email sends successfully:
      { success: true, signingUrl: '...', emailSent: true }

─────────────────────────────────────
GET /api/applications/:applicationId/offer
─────────────────────────────────────
- Return the job_offer record for this application (if exists)
- Include: all job_offer fields EXCEPT populatedContent 
  (keep that for the signing page only)
- Include offer_signatures record if signed
- Include offer_status_log entries ordered by createdAt ASC
- If no offer exists: return { offer: null }

─────────────────────────────────────
POST /api/applications/:applicationId/resend-offer
─────────────────────────────────────
- Admin or Manager only
- Fetch existing job_offer for this application
- If no offer exists or status != 'pending': return 400
- Re-send the signing link email (same template as above)
- Do NOT generate a new token — reuse the existing signingToken
- Log to offer_status_log: note = 'Offer link resent'
- Return { success: true, emailSent: boolean }
