1. ic_agreement_templates
   - id (serial primary key)
   - name (text, not null) — e.g. "Standard IC Agreement 2026"
   - content (text, not null) — full rich text/HTML body of the 
     agreement with dynamic placeholders in {{field_name}} format
   - isActive (boolean, default true) — only one should be active 
     at a time; if a new one is activated, others are deactivated
   - createdBy (integer, references staff/users table, nullable)
   - createdAt (timestamp, default now)
   - updatedAt (timestamp, default now)

   Supported placeholder fields (document these in a comment 
   above the table definition):
     {{candidate_name}}       — applicant's full name
     {{position}}             — position they are being hired for
     {{start_date}}           — their start date
     {{compensation}}         — pay rate or salary
     {{compensation_type}}    — e.g. "per hour", "per month", 
                                "flat project rate"
     {{manager_name}}         — hiring manager's full name
     {{company_name}}         — static: "The Media Optimizers"
     {{offer_date}}           — date the offer was sent
     {{custom_terms}}         — optional additional terms

2. job_offers
   - id (serial primary key)
   - applicationId (integer, references applicants/applications 
     table, not null, unique) — one offer per application
   - createdBy (integer, references staff/users table, nullable)
   - templateId (integer, references ic_agreement_templates, 
     nullable) — which template was used
   - populatedContent (text, not null) — the fully rendered 
     agreement content with all placeholders replaced with real 
     values at send time (this is the permanent record — if the 
     template changes later, this offer is unaffected)
   - compensation (text, not null)
   - compensationType (text, not null) — 'per_hour', 'per_month', 
     'flat_rate'
   - startDate (date, not null)
   - customTerms (text, nullable)
   - signingToken (text, not null, unique) — a secure random UUID 
     used to generate the public signing URL. Format: 
     /sign-offer/{signingToken}
   - status (text, not null, default 'pending') — 'pending', 
     'signed', 'declined'
   - sentAt (timestamp, default now)
   - expiresAt (timestamp, nullable) — optional expiry for the 
     signing link (nullable for now, can be used later)
   - createdAt (timestamp, default now)

3. offer_signatures
   - id (serial primary key)
   - offerId (integer, references job_offers, not null, unique) 
     — one signature per offer
   - signatureType (text, not null) — 'drawn' or 'typed'
   - signatureData (text, not null) — base64 PNG data URL if 
     drawn, or plain text name if typed
   - signerName (text, not null) — full name entered at signing
   - signerEmail (text, not null) — email confirmed at signing
   - ipAddress (text, nullable) — captured from request for 
     legal record
   - userAgent (text, nullable) — browser info for legal record
   - signedAt (timestamp, not null, default now)

4. offer_status_log
   - id (serial primary key)
   - offerId (integer, references job_offers, not null)
   - status (text, not null) — the new status value
   - changedBy (integer, nullable, references staff/users) — 
     null if changed by applicant action (e.g. signing)
   - note (text, nullable)
   - createdAt (timestamp, default now)
