3. Visibility Conditions Editor Modal

When clicking [⚙️] on a section, open a modal with:
- Section name displayed at top (read-only)
- Radio: "Always visible" vs "Show when conditions are met"
- If conditional, show:
  - Match dropdown: "AND" (all must be true) or "OR" (any must be true)
  - List of conditions, each with:
    - Question dropdown (show all questions, highlight TRIGGER questions at top)
    - Operator dropdown: "equals", "not_equals", "includes", "not_empty"
    - Value dropdown (populated based on selected question's options) or text input for non-choice questions
    - Delete condition button
  - [+ Add Condition] button
- Cancel and Save buttons

When saving, construct the visibilityConditions JSON:
{
  "operator": "AND" or "OR",
  "conditions": [
    {
      "questionId": "uuid",
      "operator": "equals",
      "value": "Option Text"
    }
  ]
}

4. API Endpoints Needed

GET /api/task-intake/sections
- Returns all sections ordered by orderIndex
- Include question count per section
- Include human-readable visibility summary string

GET /api/task-intake/sections/:id/questions
- Returns all questions for a section, ordered by orderIndex
- Include options for choice questions

PUT /api/task-intake/sections/reorder
- Body: { sectionIds: ["uuid1", "uuid2", ...] } in new order
- Updates orderIndex for all sections

PUT /api/task-intake/questions/reorder
- Body: { questionIds: ["uuid1", "uuid2", ...], sectionId: "uuid" }
- Updates orderIndex for questions within a section

PUT /api/task-intake/questions/:id/move
- Body: { newSectionId: "uuid" }
- Moves question to different section
