PART C — TRIGGER 2: HIRED STATUS →
         WELCOME EMAIL + ONBOARDING LINK
═══════════════════════════════════════

─────────────────────────────────────
Create: server/services/hiredNotificationService.ts
─────────────────────────────────────

Export a single async function:

  sendHiredNotifications(applicationId: number, 
                         changedBy: number): Promise<void>

This function must NEVER throw an unhandled error. Wrap all 
logic in try/catch — log errors and return gracefully.

Logic:

STEP 1 — Fetch application and offer details
  - Query the applications table for this applicationId
  - Join job_offers to get: startDate, compensation, 
    compensationType
  - If application not found: log warning and return
  - Extract: candidateName, candidateEmail (their personal 
    email from the application), position, startDate

STEP 2 — Determine the work email address
  - The welcome email should be sent to the hire's NEW 
    work email (@themediaoptimizers.com), not their 
    personal application email
  - Problem: we don't store the work email on the 
    applicant record — it's created externally in 
    Google Workspace
  - Solution: check if a staff record now exists in the 
    staff table where email matches a 
    @themediaoptimizers.com pattern AND the staff member's 
    name closely matches the candidate name (case-insensitive 
    first + last name match)
  - If a matching staff record is found: send to the work 
    email (staff.email)
  - If no matching staff record found: 
      - Send to the personal email (application email) as 
        fallback
      - Add a note in the notification to the manager 
        (see Step 5) that the welcome email was sent to 
        the personal email because no AgencyBoost staff 
        account was found

STEP 3 — Find and render the welcome email template
  - Search the email templates table for a template with 
    name containing 'welcome' or 'onboarding' 
    (case-insensitive) that is active/published
  - If a matching template is found: use it, substituting 
    these variables using whatever substitution pattern the 
    existing email system uses:
      {{candidate_name}} or {{name}} → candidateName
      {{position}}                   → position
      {{start_date}}                 → formatted start date
      {{onboarding_link}}            → 
        'https://agencyboost.app/onboarding'
      {{company_name}}               → 'The Media Optimizers'
  - If NO matching welcome template is found: use this 
    fallback HTML email body:

    Subject: "Welcome to The Media Optimizers, [name]! 🎉"
    
    Body (HTML):
    "Hi [candidateName],

    We are thrilled to welcome you to The Media Optimizers 
    as our new [position]!

    Your start date is [formatted start date]. Before your 
    first day, please take a few minutes to complete your 
    new hire onboarding form — it only takes a few minutes 
    and helps us get everything set up for you.

    [CTA Button — teal]: Complete Your Onboarding Form
    href: https://agencyboost.app/onboarding

    Once you log into AgencyBoost with your new work email, 
    you will also find your personalized onboarding checklist 
    ready and waiting for you under HR > Onboarding Checklist.

    We cannot wait to have you on the team. If you have any 
    questions before your start date, do not hesitate to 
    reach out.

    Welcome aboard!
    The Media Optimizers Team"

STEP 4 — Send the welcome email
  - Use the existing email sending helper (same as used 
    throughout the codebase — Nodemailer/SendGrid/etc.)
  - To: work email (or personal email fallback from Step 2)
  - Subject: from template or fallback subject above
  - Body: rendered HTML
  - On failure: log error, continue to Step 5 — do NOT 
    throw or halt

STEP 5 — Notify the hiring manager in-app
  - Find the manager: use changedBy (the staff member who 
    set the status to 'hired') as the managerId
  - Send in-app notification:
      title: "[Candidate Name] has been marked as Hired"
      message: "You have successfully hired [Full Name] 
      for the [position] role. A welcome email has been 
      sent to [email used]. Their AgencyBoost onboarding 
      checklist will be ready on their start date of 
      [formatted start date]."
      userId: managerId
  - If email was sent to personal fallback, append to 
    message: " Note: The welcome email was sent to their 
    personal email ([personal email]) because no matching 
    AgencyBoost staff account was found. Please ensure 
    their staff account is created with their work email."

STEP 6 — Fire automation trigger (if system exists)
  - Same as Part B — if an automation trigger firing 
    pattern exists, fire:
      triggerType: 'applicant_hired'
      payload: { applicationId, candidateName, position, 
                 startDate, managerId: changedBy }
  - If no automation trigger pattern: skip this step
