WHAT TO BUILD

1. Description Template System

Each section has a descriptionTemplate field (text, currently null). We'll populate these with templates that use variable substitution.

Template syntax:
- {{variable_name}} - replaced with answer value (uses question's internalLabel)
- {{#if variable_name}}content{{/if}} - conditional block, only shows if variable has value
- {{#each variable_name}}content{{/each}} - for multi-choice, loops through selected values

3. Template Engine Function

Create a function to process templates:

generateDescription(sections, answers) returns string

Logic:
1. Get all visible sections (from submission's visibleSectionIds)
2. For each visible section in order:
   a. Get the section's descriptionTemplate
   b. If template is null or empty, skip
   c. Process template with answer values
3. Concatenate all processed templates with newlines between
4. Clean up output (remove multiple consecutive blank lines)
5. Return final description string

Template processing rules:
- Replace {{variable_name}} with the answer value for the question that has that internalLabel
- You need to look up questionId by internalLabel to get the answer from the answers object
- Handle {{#if variable_name}}content{{/if}} blocks - only include content if variable has a truthy value
- Handle {{#each variable_name}}content{{/each}} for arrays (multi-choice answers)
- If a variable is undefined, null, or empty string, replace with empty string
- For multi-choice arrays, join values with ", "
- After processing, clean up any leftover empty lines or orphaned labels
