RECURRING TASK SERVICE (server/recurringTaskService.ts):

Audit the full file before changing anything. Understand the exact pattern used for the existing 24-hour job before writing any new code.

Add a new function processOnboardingWeekReleases() that runs inside the same 24-hour loop, after existing recurring task processing.

Logic:
1. Query all clients where onboardingStartDate IS NOT NULL
2. For each client, calculate daysSinceStart = Math.floor((Date.now() - client.onboardingStartDate.getTime()) / (1000 * 60 * 60 * 24))
3. Calculate currentWeek = Math.floor(daysSinceStart / 7) + 1
4. If currentWeek > client.onboardingWeekReleased:
   a. For each week from (onboardingWeekReleased + 1) through currentWeek:
      - Call generateTasksFromTemplates() with generationType "onboarding" and targetWeek = that week number
      - Pass the client's current products/bundles/packages as the items (use the same lookup pattern already in the recurring service)
   b. Update the client's onboardingWeekReleased to currentWeek
5. If currentWeek <= client.onboardingWeekReleased, skip — already released

Use a try/catch per client so one failure doesn't block others. Log successes and failures the same way the existing job logs.

Do NOT modify the existing recurring task logic. Only add the new function call at the end of the loop.
