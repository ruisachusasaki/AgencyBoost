PART B — AUTO-SYNC SERVICE
═══════════════════════════════════════

Create a new file: server/services/onboardingLmsSyncService.ts

Export a single async function:

  syncLmsCompletion(staffId: number, courseId: number): Promise<void>

Logic:

STEP 1 — Find active onboarding instance for this staff member
  - Query onboarding_instances where staffId = staffId AND 
    status = 'active'
  - If none found: return early (this staff member has no active 
    onboarding — nothing to sync)

STEP 2 — Find matching incomplete checklist items
  - Query onboarding_instance_items where:
      instanceId = instance.id
      AND itemType = 'training_course'
      AND referenceId = courseId
      AND isCompleted = false
  - If none found: return early (no checklist items link to 
    this course, or already completed)

STEP 3 — Mark matching items complete
  - For each matching item, update onboarding_instance_items:
      isCompleted = true
      completedAt = now()
      completedBy = staffId  (the hire themselves, auto-action)
      autoCompleted = true
  - Perform all updates in a single transaction

STEP 4 — Check for full instance completion
  - After marking items complete, query onboarding_instance_items 
    for this instance:
      count of isRequired = true AND isCompleted = false
  - If count = 0 (all required items done):
      update onboarding_instances set status = 'completed', 
      completedAt = now() where id = instance.id

STEP 5 — Send in-app notification to the hire
  - Use the same notification pattern established in 
    onboardingSpawnService.ts (Prompt 3)
  - Notification:
      title: "Onboarding item auto-completed"
      message: "Your training completion was detected and 
      '{courseTitle}' has been marked complete on your 
      onboarding checklist."
      userId: staffId
  - Fetch courseTitle from the LMS courses table using courseId
  - If notification system unavailable, console.log and skip

STEP 6 — If instance was just completed (Step 4 triggered):
  - Send an ADDITIONAL notification to the hire:
      title: "🎉 Onboarding complete!"
      message: "You've completed all required items in your 
      onboarding checklist. Great work!"
      userId: staffId
  - Also notify the hire's manager:
      - Find the manager: query staff table for a user where 
        teamId = hire's teamId AND role = 'manager' (or however 
        managers are identified — check existing pattern)
      - If manager found, send notification:
          title: "[Hire Name] completed onboarding"
          message: "[First Last] has completed all required items 
          in their onboarding checklist."
          userId: managerId

  The function must NEVER throw an unhandled error.
  Wrap all logic in try/catch — on error, log and return.

PART E — MANUAL SYNC ENDPOINT (ADMIN)
═══════════════════════════════════════

Add a utility route for admins to manually trigger a sync 
in case of edge cases or missed events:

POST /api/onboarding/instances/:instanceId/sync-lms
  - Admin only
  - For each onboarding_instance_item in this instance where:
      itemType = 'training_course'
      AND referenceId IS NOT NULL
      AND isCompleted = false
    Query the LMS completions table to check if staffId has 
    a completion record for that courseId
    If a completion exists: call syncLmsCompletion(staffId, courseId)
  - Returns: 
    { 
      synced: number,    -- count of items that were auto-completed
      message: string 
    }
  - Use case: a hire completed a course before the onboarding 
    checklist existed, or a sync was missed due to a bug

Add a "Sync LMS Completions" button in OnboardingInstanceDrawer.tsx:
  - Only visible to Admins
  - Small outline button in the drawer header area
  - Icon: RefreshCw from lucide-react
  - On click: calls POST /api/onboarding/instances/:id/sync-lms
  - On success: show toast "Synced X training completions" and 
    invalidate the instance query to refresh the drawer
  - On error: show destructive toast
